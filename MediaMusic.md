# 需求规格说明书 (PRD)：本地专业音乐播放器 (Advanced Local Music Player)

**文档状态:** 初始定稿

**更新日期:** 2026-06-22

## 1. 产品概述

### 1.1 产品定位
一款基于 C# 与 Web 混合技术栈开发的轻量级、高性能本地无损音乐播放器。系统专注于为用户提供纯粹的无损音频解码、极低延迟的无缝播放体验、灵活的双模曲库管理机制，以及通过 AI 辅助实现的高还原度、沉浸式桌面交互体验。

### 1.2 核心应用场景
* **发烧级伴听：** 本地高品质音乐收藏与 0 延迟无缝播放。
* **灵活管理：** 支持轻量级文件/文件夹拖拽即播，以及重度全盘扫描与 SQLite 数据库检索。
* **沉浸式交互：** 迷你模式、全透桌面歌词及系统级全局快捷键控制。

## 2. 核心功能需求 (Functional Requirements)

### 2.1 音频引擎与播放控制 (P0)
* **无损与有损格式支持：**
  * 完美解码 FLAC、APE、WAV（包含内嵌封面与元数据的精准提取）。
  * 兼容 MP3、AAC、OGG 等主流有损格式。
* **专业播放控制引擎：**
  * **无缝播放 (Gapless Playback)：** 利用底层双缓冲或多实例交替预加载机制，确保概念专辑或切歌时的音频过渡达到 0 毫秒延迟，无爆音或断流。
  * **淡入淡出 (Crossfade)：** 在播放、暂停、手动切歌时，执行线性的音量增益渐变过渡。过渡时长对用户开放自定义配置（范围：0ms - 5000ms）。
  * **多段均衡器 (EQ)：** 提供 10 段以上的频率增益调节（-12dB 至 +12dB），并支持用户保存、读取自定义频段预设（如：摇滚、低音增强、古典）。

### 2.2 本地音乐库与数据管理 (P1)
* **双模曲库管理：**
  * **轻量拖拽模式：** 用户可将外部音频文件或整个文件夹直接拖入软件歌单区域，实现即加即播，不强制写入本地数据库。
  * **深度扫描模式：** 支持指定本地多级目录。系统在后台异步进行深度扫描，提取路径与元数据，构建结构化的 SQLite 本地数据库，支持依专辑、歌手、流派的高速交叉检索。
* **元数据管理与独立编辑：**
  * 自动解析并展示音频文件（ID3 等标准）标签信息。
  * **内建元数据编辑器：** 针对原创或缺失标签的独立音频，提供可视化手动修改面板（歌名、歌手、专辑）。支持上传本地图片作为自定义封面，修改结果需同步回写至原始音频文件头部及本地 SQLite 数据库。

### 2.3 歌词解析与展示 (P2)
* **LRC 歌词精准解析：** 自动探测并匹配播放文件同目录下的同名 `.lrc` 文件，精确解析毫秒级时间戳。
* **双模歌词展现：**
  * **主界面滚动视图：** 随播放进度条同步滚动，动态高亮当前活动行，非活动行视觉降噪处理。
  * **桌面歌词悬浮窗：** 提供独立、置顶、透明、支持鼠标事件穿透的纯净桌面歌词窗口。支持文字大小、字体家族、对齐方式及颜色的全自定义。

## 3. UI/UX 与交互体验 (UI/UX Requirements)

### 3.1 窗口与多视图模式
* **全自定义无边框窗口 (Chromeless Window)：** 摒弃系统原生窗体外框，采用彻底的无边框设计，包含自定义的最大化、最小化、关闭按钮逻辑及全局拖拽阴影。
* **双轨主题系统：** 界面配色需系统性支持浅色模式 (Light) 与深色模式 (Dark) 的实时无缝切换。
* **多形态视图支持：**
  * **主界面模式：** 承载完整的图库管理、均衡器调节面板、详细歌曲信息与播放列表视图。
  * **迷你播放器 (Mini Player)：** 精简版悬浮小窗口，仅保留专辑封面、核心当前曲目滚动信息及基础播放/切歌控制按键，专为屏幕伴听优化。

### 3.2 交互控制体系
* **系统托盘常驻 (System Tray)：** 最小化或点击关闭（基于用户偏好）时隐藏至系统托盘。右键托盘图标可唤出快捷播放、退出等上下文菜单。
* **全局快捷键 (Global Hotkeys)：** 基于系统底层的键盘钩子，支持在失去焦点或被其他全屏应用覆盖时，通过组合键实现播放/暂停、上一首/下一首、音量加减等核心控制。

## 4. 技术架构说明 (Technical Architecture)

* **核心语言与运行时：** C# / .NET 8 (或后续 LTS 版本)
* **前端呈现层 (UI & 容器)：**
  * **容器化方案：** Photino.NET。利用系统原生 WebView 渲染，极低内存占用，大幅优化应用体积与冷启动速度。
  * **UI 技术栈：** HTML5 + Tailwind CSS + Vanilla JS / Blazor。设计链路：由 Figma 产出高保真设计稿后，通过 MCP (AI 代理) 1:1 还原前端代码，确保复杂动效（如毛玻璃、无边框渲染、流畅动画）的精准落地。
* **音频底层处理引擎：** ManagedBass (BASS 音频引擎的 C# 封装)
  * 原生支持 FLAC/APE/WAV 等无损编码解析。
  * 底层硬件级支持 0 毫秒无缝播放 (Gapless)、混音、交叉淡入淡出及 DSP/EQ 信号处理。
* **数据持久化与 ORM：** SQLite + Dapper
  * 摒弃重型 ORM 框架，采用轻量级 Micro-ORM Dapper。提供贴近原生 SQL 的查询性能，确保在载入与检索 5000+ 首歌曲的超大本地曲库时，主线程（UI）维持绝对流畅。
* **跨层通信机制：**
  * 采用强解耦架构。C# 后端主导文件 I/O 扫描、音频流调度及数据库事务；Web UI 前端专注视图渲染与交互。两端通过 Photino 提供的本地内存消息总线（Message Bus）完成低延迟双向数据通讯。
